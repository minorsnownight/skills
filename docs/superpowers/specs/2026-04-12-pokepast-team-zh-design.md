# PokePast Team Chinese Translation + Strategy Analysis

## Overview

A Claude Code Skill that takes one or more PokePaste URLs, translates the team data into official Chinese Showdown format, and provides detailed competitive strategy analysis.

## Architecture

Two-layer design: **Python script** for translation, **Claude Code Skill** for strategy analysis.

### Layer 1: `pokepast-translate.py`

A Python script that fetches PokePaste pages, parses team data, and outputs Chinese Showdown format text.

**Input:** One or more PokePaste URLs (command-line arguments)

**Execution flow:**
1. For each URL, fetch the PokePaste page via HTTP (no CORS proxy needed in CLI)
2. Parse HTML to extract:
   - Title (keep English)
   - Team code: 10-character alphanumeric string at the end of the title, if present
   - For each Pokemon: name, item, ability, level, EVs, IVs, nature, moves
3. Load the 7 dictionary files from `site/dict/` (pokemon, pokemon-forms, moves, abilities, items, types, natures)
4. Translate all terms to official Chinese (zh-hans, fallback to zh-hant)
5. Output formatted text

**Output format:**

```
=== Title Here TEAMCODE123 ===
队伍码: TEAMCODE123

宝可梦中文名 @ 道具中文名
特性: 特性中文名
等级: 50
努力值: 31 HP / 1 攻击 / 10 防御 / 23 特防 / 1 速度
性格: 性格中文名
- 招式中文名
- 招式中文名
- 招式中文名
- 招式中文名

下一只宝可梦 @ ...
...

=== Next Team Title ===
...
```

**Format rules:**
- Title: keep English original
- Team code: display `队伍码: XXXXXXXXXX` only if a 10-char alphanumeric code is found at the end of the title; otherwise omit this line
- All terminology in official Chinese: Pokemon names, moves, abilities, items, types, natures
- Labels unified as `标签: 值` format: `特性:`, `等级:`, `努力值:`, `个体值:`, `性格:`
- Stat abbreviations translated: Atk→攻击, Def→防御, SpA→特攻, SpD→特防, Spe→速度 (HP stays HP)
- Moves prefixed with `- ` (dash + space)
- Teams separated by `=== Title ===` headers

**Dictionary lookup:**
- Exact match first, then case-insensitive fallback
- zh-hans preferred, zh-hant as fallback if zh-hans is missing
- If no translation found, keep the English original

**HTML parsing:**
- Reuse the same parsing logic as `site/app.js`:
  - Team title from `<title>`
  - Each Pokemon from `<article>` elements
  - Name and item from first line of `<pre>` (split by `@`)
  - Ability, Level, EVs, IVs, Tera Type from labeled lines
  - Nature from `xxx Nature` pattern
  - Moves from `- xxx` lines
  - Type class from `type-xxx` CSS class (not translated, used for display only)

**Error handling:**
- If a URL fetch fails, print error and continue to next URL
- If no team data found in a page, print warning and continue

### Layer 2: Claude Code Skill `pokepast-team-zh`

**Trigger:** User invokes `/pokepast-team-zh` with one or more PokePaste URLs

**Execution flow:**
1. Call `python pokepast-translate.py <urls>` to get the Chinese Showdown text
2. Based on the translated team information, Claude provides detailed strategy analysis including:
   - **核心战术**: The team's core win condition
   - **联防关系**: Type synergy and mutual coverage between Pokemon
   - **选出建议**: Recommended lead + back row combinations for different opponent archetypes
   - **每只宝可梦的角色定位**: Role of each Pokemon and key move usage
3. Output the complete result combining translation and analysis

**Skill file location:** `.claude/skills/pokepast-team-zh/skill.md`

**Skill instructions:**
- Run the Python script first to get the translation
- Then analyze the team strategy based on the translated output
- Write the analysis in Simplified Chinese
- Output format: translation first, then strategy analysis section

## Dependencies

- Python 3 (no external packages needed, stdlib only: `json`, `os`, `sys`, `urllib.request`, `re`)
- Existing `site/dict/*.json` dictionary files (generated by `pokemon-chinese-dict.py`)
- Internet access to fetch PokePaste pages
